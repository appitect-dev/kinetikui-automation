================================================================================
WORKFLOW
================================================================================

1. CREATE: User selects template & fills details (Dashboard or API)
2. QUEUE: Video added to BullMQ render queue
3. RENDER: Remotion generates MP4 (1-2 min, parallel processing)
4. SCHEDULE: Auto-assigned to next available time slot
5. POST: Cron job uploads to Instagram at scheduled time
6. TRACK: Metrics collected via Graph API

Status Flow:
pending → rendering → rendered → scheduled → posted

================================================================================
DEPLOYMENT INSTRUCTIONS
================================================================================

Development (Local):
1. Install dependencies (npm install in each folder)
2. Setup Redis (sudo systemctl start redis-server)
3. Configure .env files (see ENV_TEMPLATE)
4. Start backend: cd backend && npm run dev
5. Start dashboard: cd dashboard && npm run dev
6. Access: http://localhost:3001

Production (VPS + Vercel):
1. Backend → VPS (46.62.209.17)
   - rsync project to VPS
   - npm install && npm run build
   - PM2: pm2 start dist/index.js
   - Nginx reverse proxy (optional)

2. Dashboard → Vercel
   - vercel --prod
   - Set BACKEND_URL env var
   - Auto-deploy from Git (optional)

3. Configure Instagram API
   - Follow INSTAGRAM-API-GUIDE.md
   - Enter credentials in Settings page
